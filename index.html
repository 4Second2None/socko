A lightweight Scala web server powered by Netty networking and AKKA processing.

We designed MashupBots to be an HTML5 style app.

Our fat html/css/javascript client will be served as static files from the web server. Server side HTML templating won't be required, it will all happen with browser javascript.

Our fat client will communicate with the business logic via a HTTP REST based API served off the same web server.

We couldn't find a lightweight asynchronous web server to meet our requirements, so we decided to write one.

We hope you find it as useful as we do.

What is Socko?

Socko is a lightweight web server that is written in Scala, runs on top of the asynchronous event driven Netty framework, and processes requests using AKKA actors.

Getting the Source

Get the source code from github

$ git clone git@github.com:mashupbots/socko.git

Out of the box, Socko supports

  • HTTP and WebSockets
  • TLS/SSL
  • Decoding HTTP POST and file uploads
  • HTTP compression
  • Routing like Unfilted HTTP ToolKit and Play Mini
  • Serving of static files

What Socko is NOT

  • Socko isn't a servlet container. We use Netty instead.
  • Socko isn't a web application framework like Lift or Play and doesn't do server side HTML templating. Socko uses client side javascript libraries like EmberJS and BackboneJS instead.
  • Socko does not store sessions or cache application data. We use HTML5 local storage instead.
